# 🧭 MoneyMate — 처음 보시는 분을 위한 가이드

이 문서는 **프로젝트를 처음 접한 사람**이 빠르게 이해할 수 있도록 정리한 요약입니다.

---

## 1. 이 프로젝트는 뭔가요?

**MoneyMate**는 **여행 경비를 같이 쓰는 사람끼리 정리하고, 누가 누구에게 얼마를 주면 되는지** 알아보기 쉽게 해 주는 웹 앱입니다.

- 여행 하나씩(예: 제주, 서울) **예산**을 정하고  
- **지출**(맛집, 교통비 등)을 **누가 냈는지** 적어 두면  
- **인당 쓴 금액**과 **정산 제안**(“A가 B에게 3만 원”)을 자동으로 보여 줍니다.

즉, **함께 여행 갈 때 쓰는 “경비 정리 + 정산” 앱**이라고 보시면 됩니다.

---

## 2. 화면은 어떻게 구성되어 있나요?

크게 **두 화면**만 있습니다.

```
[ 메인 화면 (홈) ]                    [ 지출 관리 화면 ]
     어디로 떠날까요?                    제주 여행 지출 관리
     ➕ 다른 여행 추가                    · 예산 설정/수정
     [제주] [서울] [유럽 여행] ...       · 같이 간 사람 추가
          │                              · 지출 추가/목록/수정/삭제
          │  카드 클릭                     · 개인별 사용 금액
          └──────────────────────────►   · 정산 제안 (누가 누구에게 얼마)
```

1. **메인(홈)**  
   - “어디로 떠날까요?” 아래에 **여행 카드**(제주, 서울, 유럽 여행 + 내가 추가한 여행)가 있습니다.  
   - 카드를 누르면 → 그 여행의 **지출 관리 화면**으로 이동합니다.  
   - **“다른 여행 추가”**로 새 여행 이름을 입력하면, 그 여행도 카드로 추가되고 해당 지출 관리 화면으로 갑니다.

2. **지출 관리 화면**  
   - **예산**: 총 예산 금액, 여행 기간(시작/끝 날짜)을 넣고 저장·수정합니다.  
   - **같이 간 사람**: 이름을 입력해 “추가”하면 멤버 목록에 들어갑니다.  
   - **지출**: 항목(예: 맛집), 금액, “누가 냈는지(지출자)”를 입력해 추가하고, 목록에서 수정·삭제할 수 있습니다.  
   - **정산 요약**: 사람별로 총 사용 금액이 나옵니다.  
   - **정산 제안**: “홍길동 → 김철수 25,000원”처럼 **누가 누구에게 얼마를 주면 되는지** 단계별로 보여 줍니다.

## 3. 프로젝트 폴더는 어떻게 생겼나요?

프로젝트는 **웹 화면(프론트엔드)** 과 **서버(백엔드)** 로 나뉩니다.

```
MoneyMate/
├── money-mate-client/    ← 화면(React 웹앱). 브라우저에서 보는 부분
├── money-mate-server/    ← 서버(Spring Boot). API + DB 접근
└── docs/                 ← 설명 문서들 (지금 보고 있는 것 포함)
```

### 3.1 화면 쪽 (money-mate-client)

| 폴더/파일 | 하는 일 (쉽게 말하면) |
|-----------|------------------------|
| **src/App.tsx** | “첫 화면”과 “지출 관리 화면” 경로를 나눠 줍니다. |
| **src/pages/HomePage.tsx** | 메인 화면(여행 카드, 다른 여행 추가) |
| **src/pages/JejuPage.tsx** | 지출 관리 화면 전체(예산, 동행자, 지출, 정산) |
| **src/components/BudgetList.tsx** | 예산 보기/고치기 블록 |
| **src/components/ExpenseForm.tsx** | 지출 “추가” 입력 폼 |
| **src/components/ExpenseList.tsx** | 지출 목록 + 수정/삭제 |
| **src/api/** | 서버에 요청 보내는 함수들(지출·예산·동행자) |
| **src/utils/settlement.ts** | “누가 누구에게 얼마” 정산 제안을 계산하는 로직 |
| **src/index.css** | 전체 공통 스타일(색, 폰트 등) |

### 3.2 서버 쪽 (money-mate-server)

| 폴더/파일 | 하는 일 (쉽게 말하면) |
|-----------|------------------------|
| **controller/** | 브라우저(클라이언트)가 보내는 요청을 받아서, 어떤 일을 할지 정해 줍니다. (지출 조회/추가/수정/삭제, 예산, 동행자 등) |
| **service/** | 실제 “비즈니스 로직”(저장/조회/수정 등)을 처리합니다. |
| **mapper/** | DB(MySQL)에 SQL로 데이터를 넣거나 읽어 옵니다. |
| **model/, dto/** | 지출(Expense), 예산(Budget), 동행자 요청/응답 같은 **데이터 형태**를 정의합니다. |
| **resources/application.properties** | DB 주소, 비밀번호, 포트 등 **설정**이 들어 있습니다. |
| **resources/mybatis/mapper/*.xml** | 테이블(expenses, budgets, travelers)에 대한 **SQL 문**이 들어 있습니다. |

처음 보시면 **App.tsx → HomePage / JejuPage → api/** 순서로 보시면 “화면이 어떻게 서버와 이어지는지”가 가장 빨리 보입니다.

## 4. 어떻게 실행하나요?

앱을 보려면 **서버**와 **화면** 둘 다 켜야 합니다.

### 4.1 필요한 것

- **Node.js** (화면용)
- **JDK 21** (서버용)
- **MySQL** (데이터 저장용). 서버 설정에서 `money_mate` DB를 쓰도록 되어 있습니다.

### 4.2 1단계: 서버 켜기

1. MySQL이 설치되어 있고, `money_mate` 데이터베이스가 있어야 합니다.  
2. `money-mate-server/src/main/resources/application.properties` 에서 DB 주소·아이디·비밀번호를 본인 환경에 맞게 수정합니다.  
3. 터미널에서:

```bash
cd money-mate-server
./gradlew bootRun
```

4. “Started ... Application” 같은 메시지가 나오면 서버가 떠 있는 것입니다.  
   - API 주소: **http://localhost:8080**

### 4.3 2단계: 화면 켜기

1. 새 터미널을 열고:

```bash
cd money-mate-client
npm install
npm start
```

2. 브라우저가 자동으로 열리거나, **http://localhost:3000** 으로 접속합니다.

3. 메인에서 여행 카드를 누르면 지출 관리 화면으로 넘어가고, 서버가 떠 있으면 지출·예산·동행자 데이터가 불러와집니다.

### 4.4 한 줄 요약

- **서버**: `money-mate-server` → `./gradlew bootRun` → 8080  
- **화면**: `money-mate-client` → `npm install` 후 `npm start` → 3000

---

## 5. 데이터는 어디에 저장되나요?

- **지출, 예산, 동행자** → **MySQL** 데이터베이스(서버가 접속하는 `money_mate` DB)에 저장됩니다.  
- **“내가 추가한 여행” 목록**(제주/서울/유럽 외에 사용자가 만든 카드) → 브라우저 **localStorage**에만 저장됩니다.  
  - 그래서 다른 기기나 다른 브라우저에서는 “추가한 여행” 카드가 안 보일 수 있습니다.
