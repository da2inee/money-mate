# ADR 002: 전역 상태 관리 라이브러리 미도입

## 상태

수락됨 (Accepted)

## 배경

프론트엔드에서 지출·예산·동행자 등 서버 데이터를 어떻게 보관할지 결정이 필요했습니다. Redux, Zustand, Jotai 등 전역 상태 라이브러리 도입 여부를 검토했습니다.

## 결정

- **전역 상태 라이브러리(Redux, Zustand 등)를 도입하지 않음.**
- 페이지/컴포넌트 단위 `useState` + `useEffect`로 서버 데이터 조회·갱신.
- 여행 카테고리 목록만 `localStorage`에 저장.

## 이유

- 앱 규모가 작고, 라우트가 `/`와 `/category/:category` 두 개뿐임.
- 서버 상태는 “현재 페이지에서 쓰는 데이터”가 명확하고, 페이지 마운트 시 한 번 조회 후 필요 시 재조회하면 충분함.
- 전역 상태를 도입하면 보일러플레이트와 학습 비용이 늘어나며, 현재 요구사항에서는 이득이 크지 않음.
- 나중에 캐시·낙관적 업데이트·다중 탭 동기화 등이 필요해지면 React Query(SWR 등) 또는 전역 상태 도입을 재검토할 수 있음.

## 결과

- 코드가 단순하고, 데이터 흐름이 “페이지 → API → setState”로 추적하기 쉬움.
- 새 개발자가 코드베이스를 이해하기 수월함.
- 서버 데이터 갱신은 “액션 후 getXXX 재호출” 패턴으로 일관됨.
