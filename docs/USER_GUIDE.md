## 1. 시작하기

### 1.1 필요한 것

- 웹 브라우저 (Chrome, Safari, Edge 등 권장)
- 인터넷 연결 (서버에 접속하기 위함)
- 서버가 실행 중이어야 합니다 (개발자/관리자가 서버를 켜 두었는지 확인)

### 1.2 접속 주소

- 보통 **http://localhost:3000** (개발 환경)
- 배포 환경에서는 관리자가 안내한 주소로 접속합니다.

---

## 2. 메인 화면

메인 화면에는 다음이 표시됩니다.

- **오늘 날짜**
- **"어디로 떠날까요?"** 제목과 한 줄 소개
- **"다른 여행 추가"** 버튼
- **여행 선택** 카드: 제주, 서울, 유럽 여행 + 사용자가 추가한 여행

**할 일**:  
정산할 여행을 카드에서 선택하거나, "다른 여행 추가"로 새 여행을 만든 뒤 해당 여행을 선택합니다.

---

## 3. 여행(카테고리) 선택

- **제주**, **서울**, **유럽 여행** 카드를 누르면 해당 여행의 **지출 관리 페이지**로 이동합니다.
- 사용자가 "다른 여행 추가"로 만든 여행도 같은 형태의 카드로 나타나며, 클릭 시 해당 여행 페이지로 이동합니다.
- URL이 `/category/여행이름` 형태로 바뀝니다 (예: `/category/jeju`).

---

## 4. 예산 설정

지출 관리 페이지 상단의 **예산** 영역에서 할 수 있는 것:

- **처음 사용 시**: 예산 금액, 여행 시작일·종료일을 입력하고 **저장** 버튼을 누릅니다.
- **이미 예산이 있을 때**:  
  - 예산 사용률(진행 바)과 남은 예산이 표시됩니다.  
  - **예산 수정** 버튼을 누르면 금액·날짜를 바꾼 뒤 다시 저장할 수 있습니다.
- **취소**를 누르면 수정 모드를 나가고, 변경 사항은 저장되지 않습니다.

---

## 5. 동행자 등록

**같이 간 사람**이 누구인지 등록해야, 나중에 "누가 누구에게 얼마" 정산 제안을 볼 수 있습니다.

- **같이 간 사람** 입력란에 이름을 입력하고 **추가** 버튼을 누릅니다.
- **함께한 멤버** 목록에 이름이 추가됩니다.
- 잘못 추가한 이름은 해당 행의 **삭제** 버튼으로 제거할 수 있습니다.

동행자 목록은 해당 여행(카테고리)에만 적용됩니다.

---

## 6. 지출 입력

**지출 내역**을 추가하려면:

1. **지출 항목**: 무엇에 썼는지 입력 (예: 맛집, 교통비, 숙소).
2. **금액 (원)**: 숫자만 입력 (예: 35000).
3. **지출자**: 누가 결제했는지 이름 입력 (동행자 목록에 있는 이름을 쓰면 정산 제안에서 구분하기 좋습니다).
4. **추가** 버튼을 누릅니다.

- **빠른 금액** 버튼(1만, 2만, 5만, 10만)을 누르면 해당 금액이 자동으로 들어갑니다.
- 지출이 추가되면 목록에 바로 반영되고, "지출이 추가됐어요" 같은 메시지가 잠깐 보일 수 있습니다.

---

## 7. 지출 수정·삭제

- **수정**: 각 지출 행의 **수정** 버튼을 누르면 창이 열립니다. 항목명, 금액, 지출자를 바꾼 뒤 **저장**하면 반영됩니다.
- **삭제**: **삭제** 버튼을 누르면 "정말 삭제하시겠습니까?" 확인 후 삭제됩니다. 삭제하면 목록과 정산 요약/정산 제안이 다시 계산됩니다.

---

## 8. 정산 요약·정산 제안

### 8.1 정산 요약 (개인 사용 금액)

- **개인 사용 금액**에 **이름별 총 지출액**이 표시됩니다.
- 지출할 때 넣은 **지출자** 이름 기준으로 합산됩니다.

### 8.2 정산 제안

- **정산 제안**에는 **"누가 누구에게 얼마를 주면 되는지"** 단계가 나옵니다.
- 예: "홍길동 → 김철수 25,000원"  
  → 홍길동이 김철수에게 25,000원을 주면 공평하게 정산된다는 뜻입니다.
- 동행자를 등록해 두고, 지출 시 **지출자**를 정확히 입력할수록 제안이 더 정확해집니다.

---

## 9. 다른 여행 추가

- 메인 화면에서 **"다른 여행 추가"** 버튼을 누릅니다.
- 창이 열리면 **여행 이름**을 입력합니다 (예: 부산, 일본, 제주도).
- **추가하기**를 누르면:
  - 해당 이름의 여행이 목록에 추가되고,
  - 곧바로 그 여행의 지출 관리 페이지로 이동합니다.
- 추가한 여행은 브라우저에 저장되므로, 다음에 접속해도 메인에 카드로 보입니다 (다른 기기나 브라우저에서는 보이지 않을 수 있음).

---

## 10. 자주 묻는 질문

**Q. 동행자를 안 넣으면 정산 제안이 안 나오나요?**  
A. 지출만 있어도 지출자 기준으로 정산 제안은 나올 수 있습니다. 다만 "함께한 멤버"를 등록해 두면 참가자 기준이 더 명확해집니다.

**Q. 예산을 안 넣어도 되나요?**  
A. 넣지 않아도 지출 입력·정산 제안은 사용할 수 있습니다. 예산을 넣으면 사용률·남은 예산을 볼 수 있습니다.

**Q. 여행 이름(카테고리)을 바꾸고 싶어요.**  
A. 현재는 "다른 여행 추가"로 새 이름을 만들고, 기존 여행은 그대로 두는 방식입니다. 이름 변경 기능은 추후 추가될 수 있습니다.

**Q. 데이터는 어디에 저장되나요?**  
A. 지출·예산·동행자 데이터는 서버(MySQL)에 저장됩니다. 추가한 여행 목록만 브라우저(localStorage)에 저장됩니다.

**Q. 서버가 꺼져 있으면 어떻게 되나요?**  
A. 지출 조회·추가·수정·삭제와 예산·동행자 기능은 모두 서버가 켜져 있어야 합니다. 서버가 꺼져 있으면 화면이 로딩되지 않거나 오류가 날 수 있습니다.
